Configuration requise pour le pilote JDBC

Pour accéder aux données d'une base de données SQL Server à l'aide du Pilote JDBC Microsoft pour SQL Server, vous devez installer les composants suivants sur votre ordinateur :

  • Pilote JDBC Microsoft pour SQL Server

  • Environnement d'exécution Java

Configuration requise pour l'environnement d'exécution Java

À partir de Pilote JDBC Microsoft pour SQL Server, la prise en charge de l'API Spec par le pilote JDBC (Java Database Connectivity) a été étendue pour inclure l'API JDBC 4.0. JDBC 4.0 a été introduit dans le cadre du Kit de développement logiciel Sun Java SE Development Kit (JDK) 6.0 et de l'environnement JRE (Java Runtime Environment) 6.0. JDBC 4.0 est un surensemble de JDBC 3.0.

Le Pilote JDBC Microsoft pour SQL Server prend en charge JDBC 3.0 et JDBC 4.0.

Lorsque vous déployez le Pilote JDBC Microsoft pour SQL Server sur des systèmes d'exploitation Windows et UNIX, vous devez utiliser les packages d'installation sqljdbc_<version>_enu.exe et sqljdbc_<version>_enu.tar.gz, respectivement. Pour plus d'informations sur le déploiement du pilote JDBC, consultez la rubrique Déploiement du pilote JDBC.

Pour garantir la prise en charge de la compatibilité descendante et des scénarios possibles de mise à niveau, le pilote JDBC comprend deux bibliothèques de classes JAR dans chaque package d'installation : sqljdbc.jar et sqljdbc4.jar.

JAR

Description

sqljdbc.jar

La bibliothèque de classes sqljdbc.jar prend en charge JDBC 3.0.

La bibliothèque de classes sqljdbc.jar requiert la version 5.0 de l'environnement d'exécution Java (JRE, Java Runtime Environment). L'utilisation de sqljdbc.jar dans JRE 6.0 lève une exception lors de la connexion à une base de données.

Le pilote JDBC ne prend pas en charge JRE 1.4. Vous devez mettre à niveau JRE 1.4 vers JRE 5.0 ou JRE 6.0 lors de l'utilisation du pilote JDBC. Dans certains cas, vous devrez peut-être recompiler votre application, car elle risque de ne pas être compatible avec JDK 5.0 ou version ultérieure. Pour plus d'informations, consultez la documentation disponible sur le site Web de Sun Microsystems.

sqljdbc4.jar

La bibliothèque de classes sqljdbc4.jar prend en charge JDBC 4.0. Elle inclut toutes les fonctionnalités de sqljdbc.jar, ainsi que les nouvelles méthodes JDBC 4.0.

La bibliothèque de classes sqljdbc4.jar requiert la version 6.0 de l'environnement d'exécution Java (JRE, Java Runtime Environment). L'utilisation de sqljdbc4.jar dans JRE 1.4 ou JRE 5.0 lève une exception.

Utilisez sqljdbc4.jar lorsque votre application doit s'exécuter sur JRE 6.0, même si elle n'utilise pas de fonctionnalités JDBC 4.0.

Notez que le pilote JDBC est conçu pour être pris en charge par toutes les principales machines virtuelles Java équivalentes de Sun ; toutefois, il est testé sur Sun JRE 5.0 ou version ultérieure.

Configuration requise pour SQL Server

Le pilote JDBC prend en charge toutes les connexions SQL Server 2005 et versions ultérieures.

Configuration requise pour le système d'exploitation

Le pilote JDBC est conçu pour fonctionner sur tout système d'exploitation prenant en charge l'utilisation d'une machine virtuelle Java (JVM). Toutefois, seuls les systèmes d'exploitation Sun Solaris, SUSE Linux, Windows Vista et Windows 7 ont été officiellement testés.

Langues prises en charge

Le pilote JDBC prend en charge tous les classements de colonnes SQL Server. Pour plus d'informations sur les classements pris en charge par le pilote JDBC, consultez Caractéristiques internationales du pilote JDBC.

Pour plus d'informations sur les classements, consultez la rubrique « Classements » dans la documentation en ligne de SQL Server.
