Nouveautés du pilote JDBC

Le Pilote JDBC 4.0 Microsoft pour SQL Server inclut plusieurs nouvelles fonctionnalités et améliorations.

Informations sur la connexion à une base de données SQL Azure

Il existe désormais une rubrique consacrée à la connexion à une base de données SQL Azure. Pour plus d'informations, consultez Connexion à une base de données SQL Azure.

Prise en charge de la haute disponibilité et de la récupération d'urgence

Le Pilote JDBC 4.0 Microsoft pour SQL Server prend en charge les connexions haute disponibilité/récupération d'urgence aux groupes de disponibilité AlwaysOn de SQL Server 2012. Pour plus d'informations, consultez Pilote JDBC pour la prise en charge de la haute disponibilité et de la récupération d'urgence.

Utilisation de l'authentification intégrée Kerberos pour se connecter à SQL Server

Le Pilote JDBC 4.0 Microsoft pour SQL Server utilise l'authentification intégrée Kerberos de type 4 pour se connecter à une base de données SQL Server. Pour plus d'informations, consultez Utilisation de l'authentification intégrée Kerberos pour se connecter à SQL Server. L'authentification intégrée Kerberos de type 2 est disponible dans les versions du Pilote JDBC Microsoft pour SQL Server antérieures à la version 4.0.

Accès aux informations de diagnostic dans le journal des événements étendus

Vous pouvez accéder aux informations du journal des événements étendus du serveur afin de connaître l'origine des échecs de connexion. Pour plus d'informations, consultez Accès aux informations de diagnostic dans le journal des événements étendus.

Prise en charge supplémentaire des colonnes éparses

Si votre application accède déjà aux données d'une table qui utilise des colonnes éparses, vous devriez constater une amélioration des performances. Pour obtenir des informations sur les colonnes (y compris les colonnes éparses), utilisez Méthode getColumns (SQLServerDatabaseMetaData). Pour plus d'informations sur les colonnes éparses SQL Server, consultez Utilisation des colonnes éparses.

La méthode isSparseColumnSet est ajoutée à la classe SQLServerResultSetMetaData pour indiquer si une colonne dans un jeu de résultats est un jeu de colonnes éparses.

Xid.getFormatId

Avant Pilote JDBC 4.0 Microsoft pour SQL Server, le pilote JDBC recevait un identificateur de format de l'application mais ne passait pas cette valeur au serveur de base de données. Au lieu de cela, le pilote JDBC passait un identificateur de format 0 au serveur de base de données.

À partir de Pilote JDBC 4.0 Microsoft pour SQL Server, le pilote JDBC passe l'identificateur de format de l'application au serveur de base de données. Pour obtenir le comportement mis à jour, assurez-vous que le fichier sqljdbc_xa.dll est mis à jour sur le serveur. Pour plus d'informations sur la copie d'une version mise à jour du fichier sqljdbc_xa.dll sur le serveur, consultez Transactions XA.

Itanium non pris en charge par les applications JDBC Driver 4.0

Les applications Pilote JDBC 4.0 Microsoft pour SQL Server ne peuvent pas être exécutées sur un ordinateur Itanium.
