Méthode getSchemas ()

Récupère les noms de schémas disponibles dans la base de données actuelle.

Syntaxe

public java.sql.ResultSet getSchemas()

Valeur de retour

Objet SQLServerResultSet.

Exceptions

SQLServerException

Notes

Cette méthode getSchemas est spécifiée par la méthode getSchemas dans l'interface java.sql.DatabaseMetaData.

Le jeu de résultats retourné par la méthode getSchemas contient les informations suivantes :

Nom

Type

Description

TABLE_SCHEM

String

Nom du schéma.

TABLE_CATALOG

String

Nom de catalogue pour le schéma.

Les résultats sont classés par TABLE_CATALOG, puis par TABLE_SCHEM. Chaque ligne inclut TABLE_SCHEM comme première colonne et TABLE_CATALOG comme deuxième.

Pour plus d'informations sur les données retournées par la méthode getSchemas, consultez la rubrique « sys.schemas (Transact-SQL) » dans la documentation en ligne de SQL Server.

Exemple

L'exemple suivant indique comment utiliser la méthode getSchemas pour retourner des informations sur le catalogue et ses noms de schémas associés dans SQL Server lorsque l'argument de connexion spécifie la base de données à utiliser.

public static void executeGetSchemas(Connection con) {
   try {
      DatabaseMetaData dbmd = con.getMetaData();
      ResultSet rs = dbmd.getSchemas();
      ResultSetMetaData rsmd = rs.getMetaData();

      // Display the result set data.
      int cols = rsmd.getColumnCount();
      while(rs.next()) {
         for (int i = 1; i <= cols; i++) {
            System.out.println(rs.getString(i));
         }
      }
      rs.close();
   } 

   catch (Exception e) {
      e.printStackTrace();
   }
}
