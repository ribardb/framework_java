Méthode close (ISQLServerResultSet)

Libère immédiatement les ressources de base de données et JDBC de cet objet SQLServerResultSet, au lieu de patienter jusqu'à ce que cette opération se produise lors de sa fermeture automatique.

Syntaxe

public void close()

Exceptions

SQLServerException

Notes

Cette méthode close est spécifiée par la méthode close dans l'interface java.sql.ResultSet.

Un objet SQLServerResultSet est automatiquement fermé par l'objet SQLServerStatement l'ayant généré lorsque cet objet SQLServerStatement est fermé, réexécuté ou utilisé pour récupérer le prochain résultat à partir d'une séquence comportant plusieurs résultats. Un objet SQLServerResultSet est également fermé automatiquement lorsqu'il est récupéré par le garbage collector.

Lors de l'exécution d'une instruction qui produit un ensemble unique et important de résultats avant uniquement, en lecture seule, un ensemble initial de lignes uniquement doit vous intéresser dans le jeu de résultats retourné. Dans ce cas, l'application peut appeler la méthode cancel de l'objet d'instruction associé avant la fermeture du jeu de résultats, afin de réduire la durée de traitement requise et d'ignorer les lignes restantes inutiles. Nous vous recommandons de faire la part entre le temps de traitement gagné et le temps, ainsi que l'aller-retour jusqu'au serveur requis pour annuler l'exécution, lorsque la décision doit être prise d'utiliser cette technique ou pas.
