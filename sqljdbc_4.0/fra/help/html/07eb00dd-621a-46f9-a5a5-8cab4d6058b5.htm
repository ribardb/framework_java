Configuration du mode d'envoi des valeurs java.sql.Time au serveur

Si vous utilisez un objet java.sql.Time ou le type JDBC java.sql.Types.TIME pour définir un paramètre, vous avez la possibilité de configurer la manière dont la valeur java.sql.Time est envoyée au serveur : en tant que type SQL Servertime ou que type datetime.

Ce scénario s'applique lors de l'utilisation de l'une des méthodes suivantes :

  • SQLServerCallableStatement.registerOutParameter(int, int)

  • SQLServerCallableStatement.registerOutParameter(int, int, int)

  • SQLServerCallableStatement.setTime

  • SQLServerPreparedStatement.setTime

  • SQLServerCallableStatement.setObject

  • SQLServerPreparedStatement.setObject

Vous pouvez configurer le mode d'envoi de la valeur java.sql.Time à l'aide de la propriété de connexion sendTimeAsDatetime. Pour plus d'informations, consultez Définition des propriétés de connexion.

Vous pouvez modifier par programmation la valeur de la propriété de connexion sendTimeAsDatetime à l'aide de SQLServerDataSource.setSendTimeAsDatetime.

Les versions de SQL Server antérieures à SQL Server 2008 ne prennent pas en charge le type de données time. Par conséquent, les applications utilisant java.sql.Time stockent généralement les valeurs java.sql.Time en tant que types de données datetime ou smalldatetime de SQL Server.

Si vous voulez utiliser les types de données datetime et smalldatetimeSQL Server lorsque vous travaillez avec des valeurs java.sql.Time, vous devez affecter à la propriété de connexion sendTimeAsDatetime la valeur true. Si vous voulez utiliser le type de données timeSQL Server lorsque vous travaillez avec des valeurs java.sql.Time, vous devez affecter à la propriété de connexion sendTimeAsDatetime la valeur false.

Sachez que lorsque vous envoyez des valeurs java.sql.Time dans un paramètre dont le type de données peut également stocker la date, les dates par défaut diffèrent selon que la valeur java.sql.Time est envoyée en tant que valeur datetime (1/1/1970) ou time (1/1/1900). Pour plus d'informations sur les conversions de données lors de l'envoi de données à un serveur SQL Server, consultez Utilisation des données de date et d'heure.

Dans la version 3.0 du pilote JDBC SQL Server, sendTimeAsDatetime a la valeur True par défaut. Dans une prochaine version, il se peut que la propriété de connexion sendTimeAsDatetime présente la valeur False par défaut.

Pour vous assurer que votre application continue à fonctionner comme prévu quelle que soit la valeur par défaut de la propriété de connexion sendTimeAsDatetime, vous disposez des possibilités suivantes :

  • Utilisez java.sql.Time lorsque vous travaillez avec le type de données time de SQL Server.

  • Utilisez java.sql.Timestamp lorsque vous travaillez avec les types de données datetime, smalldatetime et datetime2 de SQL Server.
