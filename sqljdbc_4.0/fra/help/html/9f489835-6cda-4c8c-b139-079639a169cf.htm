Méthode setResponseBuffering (ISQLServerStatement)

Définit le mode de mise en mémoire tampon des réponses pour cet objet SQLServerStatement selon String full ou adaptive, sans respect de la casse.

Syntaxe

public final void setResponseBuffering(java.lang.String value)

Paramètres

value

String qui contient le mode de mise en mémoire tampon des réponses. Le mode valide peut être l'une des chaînes non sensibles à la casse suivantes : full ou adaptive.

Exceptions

SQLServerException

Notes

adaptive spécifie une mise en mémoire tampon minimale des données, le cas échéant.

full spécifie la lecture de l'intégralité du jeu de résultats à partir du serveur durant l'exécution.

« adaptive » est la valeur par défaut dans les versions 2.0 et 3.0 du pilote JDBC. « full » était la valeur par défaut des versions du pilote JDBC antérieures à la 2.0.

La méthode setResponseBuffering vous permet de remplacer la propriété String de connexion responseBuffering pour l'objet SQLServerStatement actuel. Pour plus d'informations sur l'utilisation du mode de mise en mémoire tampon des réponses, consultez Utilisation de la mise en mémoire tampon adaptative.

Si l'application fournit une valeur de paramètre non valide à la méthode setResponseBuffering, une exception SQLServerException est levée.
