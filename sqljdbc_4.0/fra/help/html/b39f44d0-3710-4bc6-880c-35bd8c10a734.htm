Travail sur des types de données (JDBC)

La fonction principale du Pilote JDBC Microsoft pour SQL Server est de permettre aux développeurs Java d'accéder aux données contenues dans les bases de données SQL Server. Pour ce faire, le pilote JDBC sert d'interface pour la conversion entre les types de données SQL Server et les types et objets en langage Java.

Pour une présentation détaillée des types de données de SQL Server et du pilote JDBC, y compris leurs différences et leur conversion en types de données en langage Java, consultez Présentation des types de données de pilote JDBC.

Afin de travailler avec les types de données SQL Server, le pilote JDBC fournit les méthodes get<Type> et set<Type> pour les classes SQLServerPreparedStatement et SQLServerCallableStatement ; il fournit également les méthodes get<Type> et update<Type> pour la classe SQLServerResultSet. La méthode que vous utilisez dépend du type de données avec lequel vous travaillez et si vous utilisez des jeux de résultats ou des requêtes.

Les rubriques de cette section décrivent l'utilisation des types de données du pilote JDBC pour accéder aux données SQL Server dans vos applications Java.
